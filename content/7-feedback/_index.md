During the **First Cloud AI Journey** program and the **Serverless AI Invoice Scanner** project, I gained many useful learning and hands-on experiences. The program helped me better understand AWS Cloud, serverless architecture, real application deployment, and teamwork in a technology environment.

Through the internship, I strengthened my technical knowledge and had the chance to work with technologies such as **Amazon S3**, **AWS Lambda**, **Amazon API Gateway**, **Amazon DynamoDB**, **Amazon Cognito**, **AWS Amplify Hosting**, **Amazon Textract**, **OpenAI API**, and **Amazon CloudWatch**.

Below are my personal reflections and feedback after participating in the program.

---

## General Evaluation

### 1. Learning and Working Environment

The learning environment in the **First Cloud AI Journey** program was suitable for students who are learning about cloud computing and AWS. The content was organized in an accessible way, starting from foundational knowledge and gradually moving toward practice and project implementation.

During the program, I felt that the learning environment was open. Learners could proactively study documents, practice step by step, and record errors they encountered. This helped me improve my self-learning and problem-solving abilities.

However, I think the program could add more group discussion or periodic review sessions so learners can share progress, ask questions, and learn from each other.

---

### 2. Support From Mentors and Admin Team

The mentors and admin team provided helpful support during the learning and project implementation process. The instructions were clear, making it easier for me to follow and practice step by step.

When I encountered AWS issues such as Lambda errors, API Gateway route problems, IAM permission issues, CORS errors, or CloudWatch Logs errors, I could rely on documentation and guidance to find solutions. This helped me understand the causes more deeply instead of only following steps mechanically.

I appreciate that mentors encouraged learners to explore, experiment, and fix issues independently. This learning style helped me improve my problem-solving mindset.

---

### 3. Relevance to My Major

The internship content was highly relevant to information technology. Topics such as backend development, APIs, databases, user authentication, file storage, data processing, and system deployment are directly related to my field of study.

The **Serverless AI Invoice Scanner** project helped me apply many learned concepts in practice, especially:

- Building backend APIs.
- Working with a NoSQL database.
- Connecting frontend with backend.
- Handling file uploads.
- Using cloud services.
- Integrating AI into a system.
- Debugging and testing APIs.

Thanks to this project, I better understood how a real system is designed, deployed, and operated on AWS.

---

### 4. Opportunities to Learn and Develop Skills

During the internship, I learned many new technical and soft skills.

For technical skills, I practiced:

- AWS Management Console.
- Amazon S3 for file storage.
- AWS Lambda for backend logic.
- Amazon API Gateway for REST APIs.
- Amazon DynamoDB for invoice data storage.
- Amazon Cognito for user registration and login.
- AWS Amplify Hosting for frontend deployment.
- Amazon Textract for extracting text from invoices.
- OpenAI API for normalizing OCR data.
- Amazon CloudWatch for checking logs and debugging errors.

For soft skills, I improved:

- Self-learning.
- Teamwork.
- Report writing.
- Project presentation.
- Error analysis.
- Personal progress management.

These skills are very useful for my future work.

---

### 5. Team Culture and Collaboration

During the group project, I realized that clear task division is very important. Each member handled a different part such as frontend, backend APIs, AI processing, deployment, testing, and documentation.

Mutual support helped the team solve problems more effectively. When one part had an issue, other members could review it together and suggest solutions.

This project helped me understand that a complete system does not depend on only one person, but requires coordination between many different roles.

---

### 6. Internship Support and Policy

The program created good conditions for students to learn and practice. The content was practical and suitable for beginners learning AWS. Step-by-step documents helped me follow the process and practice independently.

I hope the program can add:

- More direct project review sessions.
- Complete demo examples for each module.
- A checklist of common AWS errors.
- Best-practice suggestions for security and cost optimization.
- Report templates or project presentation templates.

These additions would help interns complete projects and reports more easily.

---

## Other Reflections

### What I Was Most Satisfied With

What I was most satisfied with was the opportunity to directly practice AWS services through a real project. Before the internship, I only understood cloud computing at a theoretical level. After the program, I better understood how AWS services connect to create a complete system.

In particular, the **Serverless AI Invoice Scanner** project showed me how cloud and AI can be combined to solve a practical problem.

---

### What Could Be Improved for Future Interns

I think the program could improve in several areas:

- Add an initial orientation session on how to read AWS documentation.
- Provide more sample architecture diagrams for different project types.
- Add explanations for common errors such as `AccessDenied`, CORS errors, Lambda timeout, and API Gateway route errors.
- Add a mid-term review session to check project progress.
- Add more guidance about securing API keys, IAM Roles, and API Gateway Authorizer.
- Provide a checklist before demo and before cleaning up AWS resources.

These improvements would help new interns understand faster and reduce the time spent troubleshooting.

---

### Would I Recommend This Program?

I would recommend this program to friends who want to learn about AWS Cloud, serverless architecture, and how to deploy a real project.

The program helps learners:

- Build a foundation in AWS.
- Practice with real cloud services.
- Understand how to build systems with serverless architecture.
- Have a project for reports or a portfolio.
- Improve self-learning and problem-solving skills.

However, I would also advise them to be ready for a lot of self-study because AWS has many services, and errors can be difficult to understand at first.

---

## Suggestions and Expectations

### Suggestions to Improve the Internship Experience

I have several suggestions to improve the internship experience:

- Add a clearer weekly roadmap.
- Provide more architecture explanations with diagrams.
- Add short tutorial or demo videos for important steps.
- Provide an environment setup checklist before starting the project.
- Provide an AWS project security checklist.
- Hold periodic Q&A sessions with mentors.
- Hold a project review session before final report submission.
- Provide a cleanup resources template to avoid unnecessary costs.

---

### Expectations for Future Programs

I hope to continue joining or learning from similar programs in the future. After completing this project, I want to study more deeply about:

- AWS serverless best practices.
- API security with Cognito Authorizer.
- AWS Secrets Manager.
- Amazon SQS and event-driven architecture.
- CI/CD on AWS.
- Advanced monitoring and logging.
- Cost optimization when deploying cloud systems.
- Combining AI with AWS services in real projects.

---

### Additional Feedback

Overall, the **First Cloud AI Journey** program provided a good and practical learning experience. It gave me the opportunity to apply what I learned to a specific project and understand more about cloud system deployment.

I feel this was a useful experience for information technology students, especially those who want to learn about cloud computing, backend development, serverless architecture, and AI applications.

I hope the program will continue to grow, add more practical projects, and include more advanced guidance sessions so students can learn even better.

---

## Conclusion

After participating in the program, I learned many new things and gained hands-on experience with AWS. Although I still have areas to improve, the program helped me better understand real working environments, cloud system design, and the role of each component in a technology project.

I sincerely thank the mentors, admin team, and the **First Cloud AI Journey** program for creating the conditions for me to learn, practice, and complete the **Serverless AI Invoice Scanner** project.
