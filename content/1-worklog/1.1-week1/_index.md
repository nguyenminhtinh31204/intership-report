## Orientation for 8 weeks of internship

During the 8-week AWS internship, the learning and implementation process is divided into two main phases:

| Phase | Time | Main content |
|---|---|---|
| Phase 1 | Week 1 - Week 4 | Study, read documents and practice according to the program **First Cloud Journey** |
| Phase 2 | Week 5 - Week 8 | Implement the project **Serverless AI Invoice Scanner** based on learned knowledge |

During the first 4 weeks, the focus is on building a knowledge base about AWS Cloud, core services, serverless model, security, storage, databases, and application deployment. Then, the last 4 weeks will apply knowledge to real projects.

---

## Learning objectives Week 1

The first week focuses on getting familiar with the **First Cloud Journey** program, understanding how to learn AWS documentation, and grasping the foundational concepts of cloud computing.

Main goals:

- Get acquainted with the learning path of the First Cloud Journey program.
- Understand what cloud computing is and why businesses use the cloud.
- Learn cloud deployment models such as public cloud, private cloud and hybrid cloud.
- Learn cloud service models such as IaaS, PaaS and SaaS.
- Get familiar with AWS Management Console.
- Understand the concepts of AWS Region, Availability Zone and Edge Location.
- Understand the basic roles of AWS accounts, IAM users, IAM roles and access rights.
- Take notes on basic knowledge to prepare for the next weeks of study.
